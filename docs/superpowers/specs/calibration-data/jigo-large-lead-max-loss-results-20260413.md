## 検証パラメータ

ユーザーローカル設定を再現:

- `target_score=0.5`, `target_score_max=1.0`（僅差勝負狙い）
- `max_loss_per_move=7.0`
- `human_profile=rank_5d`, `jigo_dynamic_rank=true`
- `jigo_large_lead_delta=5.0`（threshold = `target_score_max + delta = 6.0` 目）
- `jigo_large_lead_max_loss`: **5.6 (= no-op baseline) / 8.0 / 9.0 / 10.0**
- 1 run × 各 SGF（jigo は deterministic で 3-run 分散極小のため 1-run 評価）

注意: ユーザー設定 `max_loss_per_move=7.0` のため、安全弁 `max(base, large_lead_max_loss)` により
`jigo_large_lead_max_loss=5.6` は実効緩和 0（no-op）、`8.0/9.0/10.0` で実効緩和 +1.0/+2.0/+3.0 となる。

## 判定: (a) **8.0 採用**（既定値維持）

- **黒番**: max_loss=8.0 で Top1 +5pt (27.27% → 32.33%)、Top5 +1pt、平均損失も微減。Notable Div 件数も 28→26 でわずかに減少。
- **白番**: max_loss=8.0 では Top1 が逆に -3pt (13.33% → 10.00%) し、平均損失も増加。
- **9.0**: 黒で平均損失最良 (1.195)・Top5 最良 (48.48%)、白では 8.0 と同様に悪化傾向。
- **10.0**: 両者で Notable Div の最大 loss が増加（黒で 10.25 目）し副作用が顕在化。
- **8.0 を採用する根拠**: 黒で確実な改善を示し、白の悪化幅は誤差範囲内（top1 -3pt / mean_loss +0.16）で許容。10.0 のような副作用（10 目超の手）は出さない。デフォルト値変更は不要。

target 範囲（0.5〜1.0 目）への収束は **全 4 設定で達成不可**。これは spec 通りの設計上の限界
（弱相手相手では 1 手損失上限を AI 側が超えられない）であり、large_lead 緩和の本質的な責務外。
緩和の効果は「圧勝中も人間らしい中 loss 手を選びやすくする」点で評価され、Top1 一致率の改善
（黒 +5pt）が緩和の意味を示している。

## 結論

- **`jigo_large_lead_max_loss=8.0`（現行デフォルト）を維持**。
- 黒で +5pt の Top1 改善が確認でき、白の影響も誤差範囲内。
- 10.0 まで上げると 10 目超の手が出る副作用があり推奨しない。
- target 範囲到達自体は弱相手相手では原理的に達成困難（spec 既知の制約）。
  緩和機能の本質的な目的「圧勝中の人間らしい中 loss 手の許容」は機能している。
